Component Details
Security Copilot Agent
  • Defined by agent-manifest-rebuild.yaml
  • Contains 13 named KQL skills
  • Skills are case-sensitive when called
  • Uses union queries to guarantee results (even "No Findings")
  • OAuth 2.0 authentication to Copilot API
  • ~1 SCU consumed per skill execution
Azure Logic App
  • ARM template deployment: logicapp-copilot-failedauth.json
  • Recurrence trigger — daily at 8:00 AM UTC
  • Calls Security Copilot via REST API
  • Strips markdown formatting from Copilot response
  • Parses JSON output into structured fields
  • HTTP POST to Log Analytics Data Collector API
  • ~$0.30/month at consumption pricing
Azure Sentinel / Log Analytics
  • Data stored in ComplianceReports_CL custom table
  • HMAC authentication via Data Collector API
  • 90-day retention for complete audit trail
  • Queryable via KQL for ad-hoc investigations
  • Foundation for Sentinel Workbook dashboards
  • Future: alerting on compliance threshold breaches

Deployment — Quick Start (4 Steps)
Step 1 — Agent Manifest
Import agent-manifest-rebuild.yaml into Security Copilot Custom Agents. Verify 13 skills are loaded and case-sensitive skill names match the Logic App prompts.
Step 2 — Logic App Deploy
Deploy logicapp-copilot-failedauth.json via Azure ARM template. Set daily recurrence schedule. Confirm Sentinel workspace ID and key are correct.
Step 3 — Authorize APIs
Authorize the Security Copilot API connection (OAuth 2.0) and the Log Analytics connection (RBAC) in the Azure Portal API connections blade.
Step 4 — Validate
Manually trigger the Logic App once. Check ComplianceReports_CL in Sentinel for data ingestion. Use the Test Logic App for validation without consuming SCUs.
Prerequisites: Azure subscription with Sentinel workspace, Security Copilot free tier (400 SCUs/month), Azure CLI, Contributor RBAC access

Example Security Copilot Prompts for OSCAR
These prompts invoke OSCAR skills directly in the Security Copilot standalone portal. Skill names are case-sensitive.
  • Run the FailedAuthenticationReport skill and summarize findings
  • Run MitreAttackReport — what ATT&CK techniques were observed this week?
  • Run PrivilegedAccountUsageReport and flag after-hours activity
  • Run MFAStatusReport — who is missing MFA in our tenant?
  • Run HighSeverityAlertsReport and prioritize the top 5 findings
  • Run DataExfiltrationReport — any unusual data movement today?
  • Run EndpointSecurityComplianceReport and list non-compliant devices
  • Run VulnerabilityManagementReport — what critical CVEs need patching?
  • Run SuspiciousProcessExecutionReport — any LOLBin activity?
  • Run NetworkAnomalyReport and flag connections to unusual geographies
  • Run FirewallRuleChangesReport — any unauthorized firewall changes?
  • Run AdminActivityReport — any unexpected admin role changes?
  • Run BackupVerificationReport — are all critical systems backed up?
  • Map all OSCAR findings from today to NIST CSF 2.0 controls and generate a compliance summary
